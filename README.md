# 仓库用途

文档记录与查阅

## 启动
```ts
npm start
```


## 打包
```ts
npm build
```



## 新增类目

```ts
新建 ./docs/新增类目/intro.md
新建 类目下的 模块

修改 docusaurus.config.ts配置
items: [
    {
        type: 'docSidebar',
        sidebarId: 'frontEndSidebar',
        position: 'left',
        label: '前端',
    },
    {
        type: 'docSidebar',
        sidebarId: 'backEndSidebar',
        position: 'left',
        label: '后端',
    },
    {
        type: 'docSidebar',
        sidebarId: 'interViewSidebar',
        position: 'left',
        label: '面试',
    },
    {
        type: 'search',
        position: 'right', // 确保搜索框位于右侧
    },
    { to: '/blog', label: '博客', position: 'left' },
    {
        href: 'https://github.com/sakuramuxia',
        label: 'GitHub',
        position: 'right',
    },
],
修改 sidebars.ts 配置

frontEndSidebar: [
    {
        type: 'autogenerated', // 自动根据文件夹生成目录结构
        dirName: 'frontEnd',
    },
],
backEndSidebar: [
    {
        type: 'autogenerated', // 自动根据文件夹生成目录结构
        dirName: 'backEnd',
    },
],
interViewSidebar: [
    {
        type: 'autogenerated', // 自动根据文件夹生成目录结构
        dirName: 'interView',
    },
]
```

## 新增博客

```ts
在 ./blog/文件夹下新增 文件 文件名最好使用 02-xxx.md 格式，一级标题作为左侧边栏的名称,二级标题作为锚点。

```

博客顶部设置 YAML Front Matter

```ts
title: Git搭建及使用
authors: [yuluo]
tags: [git]
```



## 注意格式
```ts
docu:一级标题是左侧边栏显示的标题,二级标题是右侧章节栏锚点
vite:文件名是侧边栏显示的标题,一级二级都是右侧章节栏锚点
```
